Fase 1: Configuracin y Definicin del Problema

1.1. Introduccin al Problema de Job Shop Scheduling

El Job Shop Scheduling Problem (JSSP) es un problema de optimizacin en el que un conjunto de trabajos (Jobs) debe ser procesado en un conjunto de mquinas (Machines).

Las reglas son las siguientes:

  • Cada trabajo consiste en una secuencia de operaciones con un orden de precedencia fijo.
  • Cada operacin debe ser procesada en una mquina especfica durante un tiempo determinado.
  • Una mquina solo puede procesar una operacin a la vez.

El objetivo es encontrar una planificacin que minimice el makespan: el tiempo total transcurrido desde el inicio del primer trabajo hasta la finalizacin del ltimo.

Fase 2: Modelado y Optimizacin con OR-Tools

En esta fase, traduciremos las reglas y datos del problema a un modelo matemtico que el solver de OR-Tools pueda entender y resolver.

2.1. Creacin del Modelo y Variables de Decisin

Iniciamos creando una instancia del modelo. Luego, para cada operacin, creamos una variable de intervalo (IntervalVar). Este es el objeto fundamental en problemas de planificacin: representa una tarea que tiene un inicio, un fin y una duracin fija. Guardaremos todas estas variables en un diccionario para acceder a ellas fcilmente.

In[85]:
model = cp_model.CpModel()
# 1. Calcular un horizonte de tiempo (un lmite superior seguro para el makespan)
horizon = sum(task[1] for job in jobs_data for task in job)
all_tasks = {}

# 2. Creamos las variables de intervalo para cada operacin
for job_id, job in enumerate(jobs_data):
    for task_id, task in enumerate(job):
        machine, duration = task
        suffix = f'_{job_id}_{task_id}'
        
        # Cada operacin tiene una variable para su inicio, fin e intervalo
        start_var = model.NewIntVar(0, horizon, 'start' + suffix)
        end_var = model.NewIntVar(0, horizon, 'end' + suffix)
        interval_var = model.NewIntervalVar(start_var, duration, end_var, 'interval' + suffix)
        
        all_tasks[(job_id, task_id)] = interval_var

print(f"*** Se crearon {len(all_tasks)} variables de intervalo.")

*** Se crearon 9 variables de intervalo.

2.2. Implementacin de Restricciones (Precedencia y Mquinas)

Ahora aadimos las dos reglas principales del problema:

  1. Restriccin de No Superposicin: Para cada mquina, las operaciones que la utilizan no pueden ejecutarse al mismo tiempo. La funcin AddNoOverlap se encarga de esto elegantemente.
  2. Restriccin de Precedencia: Para cada trabajo, las operaciones deben seguir su secuencia fija. La operacin i+1 no puede empezar hasta que la operacin i haya terminado.
In[86]:
# a) Restriccin de no superposicin en las mquinas
for machine_id in range(num_machines):
    intervals_on_machine = []
    for job_id, job in enumerate(jobs_data):
        for task_id, task in enumerate(job):
            if task[0] == machine_id:
                intervals_on_machine.append(all_tasks[(job_id, task_id)])
    
    # El modelo se asegura de que ninguno de estos intervalos se solapen
    model.AddNoOverlap(intervals_on_machine)

# b) Restriccin de precedencia dentro de un mismo trabajo
for job_id, job in enumerate(jobs_data):
    for task_id in range(len(job) - 1):
        op_actual = all_tasks[(job_id, task_id)]
        op_siguiente = all_tasks[(job_id, task_id + 1)]
        
        # Forzamos a que el inicio de la siguiente operacin sea >= al fin de la actual
        model.Add(op_siguiente.StartExpr() >= op_actual.EndExpr())

print("*** Resstricciones de no superposicin y precedencia.")

2.3. Definicin del Objetivo (Minimizar Makespan) y Resolucin

El ltimo paso es decirle al modelo qu es lo que queremos optimizar. Nuestro objetivo es minimizar el makespan. El makespan se define como el tiempo de finalizacin de la ltima operacin de todos los trabajos.

Una vez definido el objetivo, creamos una instancia del solver y le pedimos que resuelva el modelo.

In[87]:
# Variable objetivo: makespan
makespan = model.NewIntVar(0, horizon, 'makespan')

# Definimos el makespan como el mximo de los tiempos de finalizacin  de la ltima operacin de cada trabajo.

last_ops = [all_tasks[(job_id, len(job) - 1)] for job_id, job in enumerate(jobs_data)]
model.AddMaxEquality(makespan, [op.EndExpr() for op in last_ops])
model.Minimize(makespan)

# Inicializamos el solver 
solver = cp_model.CpSolver()
status = solver.Solve(model)

# Comprobamos el estado de la solucin
if status == cp_model.OPTIMAL or status == cp_model.FEASIBLE:
    print(f"   - Makespan ptimo Encontrado: {solver.ObjectiveValue()}")
else:
    print(" No se encontr una solucin ptima.")
